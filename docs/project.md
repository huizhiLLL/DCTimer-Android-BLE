# DCTimer-Android 项目现状

日期：`2026-05-16`

## 项目定位

`DCTimer-Android` 当前是基于原版 DCTimer 的蓝牙硬件计时增强版本，重点是把智能魔方和蓝牙计时器的真实硬件计时链路做稳。

当前硬件方向：

- 智能魔方：保障 `3x3` 蓝牙计时、打乱引导、自动起停表、成绩保存和解法重建。
- 蓝牙计时器：保障 `QiYi Smart Timer` 的准备、运行、停止、断开重连和异常恢复。

## 当前基线

- Android 工程：`AndroidX + AGP 8.9.2 + Gradle 8.11.1 + JDK 17`
- `compileSdk / targetSdk`：`35`
- 文件导入导出、打乱导入导出、背景图选择：`SAF / Uri`
- release 签名：`key.properties` 或 `-P` 参数注入

## 已接入硬件

- 智能魔方：`MoYu32`、`QiYi / Tornado V4 智能版`、`GAN v2 / v3 / v4`
- 蓝牙计时器：`QiYi Smart Timer`

## 当前能力

### 智能魔方

- 已打通“按打乱进入 `READY` -> 首转起表 -> 还原停表 -> 自动保存成绩”的主流程。
- BLE 扫描按设备结果直列展示，连接阶段自动识别协议。
- 计时页和魔方状态弹窗已接入 `GLSurfaceView + OpenGL ES 2.0` 的 `3x3` 实时 3D 预览，支持基础层转动画和手动拖动视角；普通打乱图仍使用 Canvas / Bitmap。
- 打乱体验支持进度提示、偏离后的逆序纠错、超长纠错提示先复原，以及打乱完成后的 `READY` 等待首转。
- `WCA` 观察模式下，打乱完成后进入观察，首次真实转动起表。

### 解法重建与统计

- 智能魔方成绩保存原始步序 `moves`，并可追加可空 `solve_meta`。
- 已支持 `333-smart-cf4op` 分段：`Cross / F2L 1-4 / OLL / PLL`。
- 分段边界、重建步数 TPS、`100ms` 内对向层组合识别为 `E / M / S` 已完成真机验证。
- `AUF` 默认并入 `PLL`；`U2` 与 `E / M / S` 按一步计，`x / y / z` 不计入主 TPS。
- 成绩列表支持默认“最新在下”和可选“最新在上”，该时间方向应用于全部分组；成绩、平均和历史 `PB` 排序仅作为当前分组临时排序。

### 蓝牙计时器

- `enterTime` 已拆成“智能魔方 / 蓝牙计时器”两条独立硬件入口。
- `QiYi Smart Timer` 已按独立协议接入。
- 设备语义：准备态只显示待起表，运行态由本地时钟刷新，停表后以设备成绩保存。
- 蓝牙计时器模式不启用应用内 `WCA` 观察和观察提示。
- `READY / RUNNING / STOPPED` 主链、断开、重连与异常恢复已完成真机验证。
