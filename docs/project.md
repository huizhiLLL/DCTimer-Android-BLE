# DCTimer-Android 项目现状

日期：`2026-05-02`

## 项目定位

`DCTimer-Android` 当前是基于原版 DCTimer 的蓝牙硬件计时增强版本。项目重点不再是通用计时器功能扩展，而是围绕智能魔方和蓝牙计时器，把真实硬件计时链路做稳。

当前硬件方向分为两条：

- 智能魔方：重点保障 `3x3` 蓝牙计时、打乱引导、自动起停表、成绩保存和解法重建。
- 蓝牙计时器：当前正式接入 `QiYi Smart Timer`，重点保障准备、运行、停止、断开重连和异常恢复。

## 当前基线

- Android 工程已升级到 `AndroidX + AGP 8.9.2 + Gradle 8.11.1 + JDK 17`。
- `compileSdk / targetSdk` 已提升到 `35`。
- 数据库导入导出、打乱导入导出、背景图选择已迁移到系统文档选择器和 `SAF / Uri`。
- 旧 `FileSelectorDialog`、主线 APK 自更新入口、安装权限与 `FileProvider` 已退出主线。
- release 签名链路已标准化，支持 `key.properties` 和 `-P` 参数注入签名信息。

## 已接入硬件

### 智能魔方

- `MoYu32`
- `QiYi / Tornado V4 智能版`
- `GAN v2 / v3 / v4`

### 蓝牙计时器

- `QiYi Smart Timer`

## 当前已完成能力

### 智能魔方计时主链

- 已打通“按打乱进入 `READY` -> 首转起表 -> 还原停表 -> 自动保存成绩”的主流程。
- BLE 扫描弹窗已取消旧协议类别切换，当前按扫描结果直列展示，并在连接阶段自动识别协议。
- 智能魔方状态展示已切换到自定义 3D 渲染控件。
- 打乱体验已支持进度提示、偏离后的逆序纠错、超长纠错时提示先复原，以及打乱完成后的 `READY` 等待首转。
- 开启 `WCA` 观察时，智能魔方按“打乱完成立即进入观察 -> 首次真实转动起表”工作，不再允许点击页面手动结束观察起表。
- 普通计时和智能魔方绿色准备态已统一为沉浸预览布局，显示 `0.00 / 0.000`。

### 解法重建与统计

- 智能魔方成绩已保存原始步序 `moves`。
- 已新增可空 `solve_meta` 元信息字段，兼容旧数据库和旧数据导入。
- 成绩列表已支持历史 `PB` 标记：单次、滚动平均 1、滚动平均 2 会按成绩顺序重新计算每次打破个人最好成绩的位置，旧成绩加载后也能显示过去的破 PB 记录。
- 普通成绩列表三列表头支持点击按对应列历史 `PB` 记录升序查看；同一列表头再次点击恢复正常顺序，同一时间只保留一个 PB 排序列。
- 已支持 `333-smart-cf4op` 分段展示：`Cross / F2L 1-4 / OLL / PLL`。
- `333-smart-cf4op` 分段边界和重建步数 TPS 已完成真机验证。
- `AUF` 当前不单独拆分阶段，默认并入 `PLL`。
- TPS 当前按重建后步数统计，`U2` 与 `E / M / S` 均按一步计，`x / y / z` 视角转体不计入主 TPS。
- 已支持 `100ms` 内对向层组合识别为 `E / M / S`。
- `QiYi / Tornado V4` 已接入陀螺仪 quaternion 事件，并可在手动初始化视角后参与真实视角转体和高置信宽层识别；该链路仍需继续真机验证。

### 蓝牙计时器主链

- `enterTime` 已拆成“智能魔方 / 蓝牙计时器”两条独立硬件入口。
- 旧 `SmartTimer.java` 和旧 `BLEDevice.TYPE_GAN_TIMER` 已清理。
- `BluetoothTools` 已增加 `SmartTimerProtocol` 分发支路。
- `QiYi Smart Timer` 已按独立 `QiyiSmartTimerProtocol` 接入。
- `QiYi Smart Timer` 当前按设备语义工作：
  - 准备态只显示绿色待起表，不提前跑表。
  - 起表后由本地时钟连续刷新页面。
  - 停表时以设备成绩收口显示并保存。
  - 蓝牙计时器模式下禁用应用内 `WCA` 观察和观察提示。
- 智能魔方 / 蓝牙计时器断联，或未连上设备前关闭连接弹窗时，会自动回退到普通计时器。
- BLE 首次权限授权完成后会重新进入扫描流程，避免连接弹窗先开导致首轮扫描为空。
- `QiYi Smart Timer` 的 `READY / RUNNING / STOPPED` 主链、断开、重连与异常恢复已完成真机验证。

## 当前任务

### 智能魔方重建统计待验证

- 真机验证 `QiYi / Tornado V4` 陀螺仪重建链路：
  - 连接后手动初始化视角为“白顶绿前”。
  - 验证首转前起手姿态进入 `Cross` 开头。
  - 验证真实视角转体、宽层识别、中层 `E / M / S` 不误识别为转体。
  - 验证普通 CFOP 分段，以及保存后的 `moves / solve_meta`。
- 后续如果 `MoYu32 / GAN` 接入陀螺仪事件，再扩展真实视角转体和宽层识别。

### 后续候选方向

- 线上房间对战可作为下一条智能魔方产品主线，但需要正式立项后再展开方案。
- 是否继续扩更多蓝牙计时器协议，作为新的设备接入需求单独评估。
- 如需补做异常恢复提示或手动输入 `MAC` 兜底，应作为独立子任务重新立项。
