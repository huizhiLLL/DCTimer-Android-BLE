# Android 底座升级方案（已完成）

## 1. 目标

把工程从旧的 `support library + AGP 3.6 + targetSdk 29` 升级到可继续维护、可按正式发布准备的现代基线，同时不回退智能魔方主链与现有文件能力。

## 2. 本次落地结果

- 已迁移到 `AndroidX + AGP 8.9.2 + Gradle 8.11.1 + JDK 17`
- 已将 `compileSdk / targetSdk` 提升到 `35`
- 已将数据库导入导出、打乱导入导出、背景图选择迁移到 `SAF / Uri`
- 已移除旧 `FileSelectorDialog`、根外部目录主流程、APK 自更新入口、`REQUEST_INSTALL_PACKAGES` 与 `FileProvider`

## 3. 当前实现要点

- `MainActivity` 与 `DetailActivity` 统一改为系统文档选择器处理文件导入导出
- 背景图持久化改为 `picUri`，保留旧 `picPath` 作为只读兼容回退
- BLE 权限在 Android 12+/15 上只走 `BLUETOOTH_SCAN / BLUETOOTH_CONNECT`
- 仍保留 Android 11 及以下的 `ACCESS_FINE_LOCATION / BLUETOOTH / BLUETOOTH_ADMIN` 兼容声明

## 4. 当前待继续验证项

- Android 12 / 15 真机上的数据库导入导出、打乱导入导出、背景图持久化
- Android 12 / 15 真机上的 BLE 扫描、连接、断开与自动计时回归

## 5. 2026-04-08 真机回归修正

- “计时页左下角统计信息消失”经确认是用户关闭了 `显示统计简要` 设置项，不是布局缺陷；相关误判布局改动已撤回。
- 针对应用窗口覆盖系统导航栏的问题，已保留主页面与相关页面的窗口边界修正，避免内容继续延伸到导航栏区域。
- 针对真机“扫描不到任何蓝牙设备”的问题，已恢复 `ACCESS_FINE_LOCATION` 声明与 Android 12+/15 上的运行时申请，当前 BLE 扫描权限链改为 `BLUETOOTH_SCAN + BLUETOOTH_CONNECT + ACCESS_FINE_LOCATION` 的兼容模式。
